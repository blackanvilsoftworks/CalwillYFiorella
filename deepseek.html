Ubicación de Páginas Legales

¿Dónde colocar Términos y Condiciones y Política de Privacidad?

¿En el root o en /src/pages?

Puedes colocarlas perfectamente en ./src/pages/ junto con tus otras páginas. No es necesario que estén en la raíz del proyecto.

tu-proyecto/
├── src/
│ ├── pages/
│ │ ├── auth/
│ │ ├── admin/
│ │ ├── dashboard/
│ │ ├── terminos-condiciones.html
│ │ └── politica-privacidad.html
│ ├── components/
│ └── styles/
├── public/
└── package.json

Esta estructura es completamente válida y organizada.

Opciones de organización

Opción 1: Directamente en /src/pages
src/
└── pages/
├── auth/
├── admin/
├── dashboard/
├── terminos-condiciones.html
└── politica-privacidad.html

Ventaja: Sencillo y directo.

Opción 2: En subcarpeta /legal dentro de /pages
src/
└── pages/
├── auth/
├── admin/
├── dashboard/
└── legal/
├── terminos-condiciones.html
└── politica-privacidad.html

Ventaja: Más organizado, especialmente si añades más páginas legales en el futuro.

Opción 3: En carpeta /static o /public
src/
└── pages/
├── auth/
├── admin/
└── dashboard/
static/
└── legal/
├── terminos-condiciones.html
└── politica-privacidad.html

Ventaja: Separa el contenido estático/legal de las páginas de la aplicación.

Recomendación

Te recomiendo la Opción 2: Crear una subcarpeta /legal dentro de /pages porque:

  • Mantiene la organización coherente con tu estructura actual
  • Es escalable si añades más páginas legales (como "Cookies", "Aviso Legal", etc.)
  • Mantiene separado el contenido legal del funcional
  • Es semanticamente correcto

Implementación con la opción recomendada

src/
└── pages/
├── auth/
│ ├── login.html
│ └── register.html
├── admin/
│ ├── users.html
│ └── settings.html
├── dashboard/
│ ├── index.html
│ └── profile.html
└── legal/
├── terminos-condiciones.html
└── politica-privacidad.html

Los enlaces en tu footer serían:

<footer> <a href="/src/pages/legal/terminos-condiciones.html">Términos y Condiciones</a> <a href="/src/pages/legal/politica-privacidad.html">Política de Privacidad</a> </footer>

Si estás usando Vite, puedes configurar alias para paths más limpios:

// vite.config.js import { defineConfig } from 'vite' import path from 'path' export default defineConfig({ resolve: { alias: { '@': path.resolve(__dirname, './src'), '@pages': path.resolve(__dirname, './src/pages'), '@legal': path.resolve(__dirname, './src/pages/legal') } } })

Y luego usar enlaces más limpios:

<footer> <a href="@legal/terminos-condiciones.html">Términos y Condiciones</a> <a href="@legal/politica-privacidad.html">Política de Privacidad</a> </footer>

Configuración para producción

Si estás usando un tooling de build (como Vite, Webpack, etc.), asegúrate de que:

  1. Las páginas legales se copien al directorio de build final
  2. Los enlaces funcionen correctamente en producción

En Vite, por defecto, todo en la carpeta src se procesa, pero puedes asegurarte añadiendo esto en vite.config.js:

// vite.config.js export default defineConfig({ build: { rollupOptions: { input: { main: resolve(__dirname, 'index.html'), terminos: resolve(__dirname, 'src/pages/legal/terminos-condiciones.html'), privacidad: resolve(__dirname, 'src/pages/legal/politica-privacidad.html') } } } })

O simplemente asegurándote de que tu HTML principal tenga enlaces correctos a estas páginas.

Resumen final

  • Sí, puedes poner las páginas en ./src/pages/
  • Recomiendo crear una subcarpeta /legal dentro de /pages
  • Ajusta los enlaces en tu footer para apuntar a la nueva ubicación
  • Si usas Vite, considera configurar alias para paths más limpios
  • Verifica que todo funcione correctamente en producción

Esta approach mantiene tu proyecto organizado y escalable, permitiéndote añadir fácilmente más páginas legales en el futuro si es necesario.
